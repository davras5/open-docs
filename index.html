API Documentation

Draft v0.1

This is a proposed REST API for OpenDocs. Endpoints are designed to expose the current application capabilities as a backend service. Authentication, pagination, and error schemas are not yet defined.

Base URL https://api.opendocs.example/v1

Documents

GET /documents List all documents
Query folderId search tags status sort limit offset
Response 200 { "documents": [...], "total": 28, "limit": 50, "offset": 0 }
GET /documents/{id} Get document metadata
Response 200 { "id", "title", "creator", "status", "revision", "versions", ... }
GET /documents/{id}/content Download file binary
Response 200 Binary file stream (Content-Type from metadata)
POST /documents Upload a new document
Body multipart/form-data: file, folderId, title, description, tags[], discipline, phase
Response 201 { "id": "doc-xxx", "slug": "...", "shareToken": "..." }
PUT /documents/{id} Update document metadata
POST /documents/{id}/versions Upload a new version
Body multipart/form-data: file, comment
DELETE /documents/{id} Move document to trash (soft delete)

Folders

GET /folders List all folders
Query parentId
POST /folders Create a folder
PUT /folders/{id} Rename or move folder
DELETE /folders/{id} Delete folder and contents

Sharing

GET /share/{token} Resolve share link to document
POST /documents/{id}/share Create or update share link
Body { "email": "...", "permission": "view|edit" }
DELETE /documents/{id}/share/{email} Revoke access

Search

GET /search Full-text search across documents and metadata
Query q folderId type tags discipline status limit

Metadata & Project

GET /project Get project metadata
GET /tags List all tags with document counts
GET /disciplines List disciplines (Architektur, Bauleitung, ...)
